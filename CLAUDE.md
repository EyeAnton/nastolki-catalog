# Boardex.am — проект

Личный каталог б/у настольных игр в Армении. Идея: ротация игр между людьми, обмен и продажа. Без монетизации. Личная инициатива @EyeAnton.

**Сайт:** https://nastolki-catalog.pages.dev  
**Telegram-канал с объявлениями:** https://t.me/used_boardgames_armenia  
**GitHub:** https://github.com/EyeAnton/nastolki-catalog

---

## Стек

- Один файл `index.html` — весь фронтенд (HTML + CSS + JS встроены)
- Обложки: `covers/{bgg_id}.webp` (300×300, WebP quality=82)
- Деплой: Cloudflare Pages через `deploy.ps1`
- Никакого фреймворка, никакого билда — просто статика

## Деплой

```powershell
# Токен хранится в ~/.nastolki-token (не в репо!)
powershell -File deploy.ps1
```

Скрипт сам читает токен, собирает `dist/` и деплоит через wrangler. **Запускай сам после каждого изменения — не жди пока пользователь попросит.**

## Добавление игры

Используй skill `/add-game` — он проведёт по всем шагам.

Порядок шагов:
1. Найти точный BGG ID через WebSearch (`site:boardgamegeek.com "название" boardgame`) — **никогда не угадывать**
2. Скачать обложку: Tesera.ru первым, Yandex Images как fallback
3. Добавить запись в BGG (если игры ещё нет)
4. Добавить листинг в D
5. Проверить что `D[i].t` точно совпадает с ключом в BGG
6. Git commit + push
7. Задеплоить через `deploy.ps1`

## Правила

| Правило | Детали |
|---|---|
| Имена продавцов | Не отображать — только ссылка на Telegram-сообщение |
| BGG ID | Всегда проверять через WebSearch, никогда не угадывать |
| Обложки | Tesera.ru (точнее) → Yandex Images (fallback) |
| Обмен | Поле `x: true` в D-записи — показывает бейдж ⇄ и попадает в фильтр |
| `t` в D | Должен точно совпадать с ключом в BGG, иначе карточка без обложки и рейтинга |
| Удаление из каталога | По просьбе продавца — удалить D-запись, BGG-запись оставить |

## UI-особенности

- Клик на обложку → модальное окно с полной карточкой
- BGG-бейдж на карточке → ссылка на BGG
- «Найти продавца» → открывает Telegram-сообщение напрямую
- Каждые 9 карточек — вдохновляющая цитата про ротацию игр
- Первый визит — онбординг-тултип с объяснением интерфейса
- Адаптивная сетка: 2 колонки (мобайл) → 3 → 4 → 5 (широкий)
